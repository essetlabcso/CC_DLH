# Annex 4: CSO Capacity Taxonomy and Indicator Mapping

## DEC Learning Hub Course Creator Portal

## 1. Purpose of This Annex

This annex defines the **CSO Capacity Taxonomy and Indicator Mapping** for the DEC Learning Hub Course Creator Portal.

Its purpose is to give Codex/GPT-5.5, developers, course creators, reviewers, and DEC stakeholders a clear, controlled reference for how courses should be linked to CSO capacity areas, sub-capacities, indicators, standards, practical learning objectives, final tests, practical proof, verified achievements, and monitoring dashboards.

This annex should help the platform prevent generic course creation by ensuring that every course is traceable to:

> **CSO capacity area → sub-capacity → diagnosed gap → indicator/standard → target learner group → performance goal → course design → final test → optional practical proof → verified achievement → monitoring evidence**

The DG NEAR Guidelines are useful because they distinguish three broad areas: enabling environment, CSO–public authority cooperation, and CSO capacity/resilience. For the DEC Course Creator Portal, the most directly course-relevant area is **Specific Objective 3: CSO capacity and resilience to carry out activities effectively**, while enabling-environment issues should usually be routed to non-course or blended support pathways unless a separable Knowledge/Skill component exists.

# 2. Source Frameworks Used

This taxonomy is adapted from the following uploaded and developed sources:

1.  **DG NEAR Guidelines for EU Support to Civil Society 2021–2027**
Especially Specific Objective 3, which includes indicators on governance, communication of results, transparency, M&E, evidence use, partnerships, diversified funding, and human resources.

2.  **Global Standard for CSO Accountability: 12 Commitments for Dynamic Accountability**
This provides accountability commitments across what CSOs want to achieve, how they work, and what they do internally. It emphasizes stakeholder dialogue, feedback, benchmarks, baselines, monitoring, self-regulation, verification, and targeted capacity support.

3.  **Civil Society Assessment Report 2020 / EU TACSO-related assessment resources**
These resources reinforce the practical importance of transparency, information sharing, M&E, strategic planning, funding sustainability, evidence use, and gender mainstreaming as recurring capacity concerns.

4.  **DEC Learning Hub refined Analysis, Design, Build, Review/Publish, and M&E specifications**
These define how the taxonomy should operate inside the portal as controlled dropdowns, handover fields, review checks, catalog tags, proof links, and monitoring categories.

This annex is not intended to copy international frameworks mechanically. It adapts them into a practical, locally usable taxonomy for Ethiopian local CSOs and the DEC Learning Hub.

# 3. Core Taxonomy Principle

The taxonomy should not be a decorative list of topics. It should function as a **controlled workflow backbone**.

The platform should use the taxonomy in:

- Analysis Gate;

- Master Capacity Gap Bank;

- Capacity Map;

- Action Map;

- Learning Design Document;

- Storyboard and Block Plan;

- Build Studio block metadata;

- Review checklist;

- Publish metadata;

- Learner course catalog;

- certificate context;

- practical proof configuration;

- verified achievement/badge records;

- monitoring dashboards.

The key product rule is:

> A course should not be created or published unless it is linked to a recognized CSO capacity area and a specific diagnosed gap.

# 4. Taxonomy Levels

The platform should use four linked levels.

| **Level**                          | **Description**                       | **Example**                                                                            |
|------------------------------------|---------------------------------------|----------------------------------------------------------------------------------------|
| Level 1: Capacity Area             | Broad CSO capacity domain             | Financial Management and Resource Mobilization                                         |
| Level 2: Sub-Capacity              | More specific capacity focus          | Donor budget justification                                                             |
| Level 3: Diagnosed Gap             | Concrete performance/capacity problem | Program staff cannot clearly link budget lines to activities and allowable costs       |
| Level 4: Indicator / Standard Link | External or internal reference        | DG NEAR SO 3.7: CSOs have diversified funding base; internal donor readiness indicator |

This structure keeps the system practical. It avoids both extremes: too broad to guide course creation and too detailed to become unmanageable.

# 9. Enabling Environment vs Course-Addressable Capacity

The platform must distinguish between **external enabling-environment barriers** and **course-addressable Knowledge/Skill gaps**.

This is important because DG NEAR includes enabling environment objectives such as freedom of association, assembly, expression, funding, taxation, public financial support, and volunteering/employment policy. These are important, but many are not solvable through a course alone.

| **Issue type**                       | **Example**                                            | **Platform decision**                                                                     |
|--------------------------------------|--------------------------------------------------------|-------------------------------------------------------------------------------------------|
| Enabling environment / legal barrier | Restrictive law prevents CSO registration              | Record and route outside course workflow unless legal-awareness K/S component exists      |
| Public authority practice barrier    | CSOs excluded from formal consultations                | May support advocacy skills, but systemic barrier remains non-course                      |
| Organizational capacity gap          | Staff cannot document consultation inputs clearly      | Course-addressable Skill gap                                                              |
| Knowledge gap                        | CSO staff do not know consultation timeline or process | Course-addressable Knowledge gap                                                          |
| Digital access barrier               | Rural staff cannot access internet                     | Environment gap; not course-addressable unless offline reporting skill is separable       |
| Motivation/fear barrier              | Staff fear advocacy consequences                       | Not course-addressable alone; may need safe advocacy judgment course plus broader support |

The platform should treat enabling-environment issues as context, not automatically as course topics.

# 14. Indicator Strength Rating

The platform may include a simple rating for how strong the indicator linkage is.

| **Rating**           | **Meaning**                                                                          |
|----------------------|--------------------------------------------------------------------------------------|
| Strong               | Directly linked to a recognized DG NEAR SO 3 indicator or Global Standard commitment |
| Moderate             | Linked to a recognized standard but adapted for local/DEC context                    |
| Emerging             | Locally important capacity area with no exact external indicator                     |
| Internal             | DEC-specific capacity or digital skill indicator                                     |
| Non-course / context | Important issue, but mainly enabling-environment or systemic                         |

This helps reviewers evaluate whether a proposed course is strategically grounded.

# 15. Course-Fit Decision Rules by Capacity Area

Each capacity area can include Knowledge, Skill, Motivation, and Environment issues. Phase 1 courses should only proceed where there is a clear Knowledge or Skill component.

| **Capacity area**             | **Course-addressable examples**                                | **Non-course / blended examples**                           |
|-------------------------------|----------------------------------------------------------------|-------------------------------------------------------------|
| Governance                    | Learn board roles; practice conflict-of-interest documentation | Board unwilling to meet; leadership conflict                |
| Accountability                | Categorize feedback; draft response note                       | Communities fear retaliation; no complaint mechanism exists |
| Strategic Planning            | Translate strategy into action plan                            | No leadership commitment to strategy                        |
| Finance/Resource Mobilization | Prepare budget justification; map donors                       | Legal restrictions on foreign funding                       |
| HR/Safeguarding               | Apply referral pathway; use onboarding checklist               | No safeguarding focal point appointed                       |
| Advocacy                      | Draft evidence-based policy message                            | Public authorities refuse consultation                      |
| MEAL                          | Use outcome indicator; write evidence statement                | No time or budget for data collection                       |
| Digital Skills                | Use spreadsheet; protect sensitive data                        | No internet/device access                                   |
| Partnerships                  | Prepare role matrix; clarify joint action                      | Partners refuse equitable collaboration                     |

This rule protects the platform from forcing systemic barriers into digital course format.

- preserve the selected capacity area across Analysis, Design, Build, Publish, and Monitoring;

- use the selected capacity area to label course cards and catalog entries;

- require a capacity area before Analysis can lock;

- require capacity linkage before Publish;

- link final test and proof records to the course’s capacity area;

- expose capacity area in monitoring aggregation;

- avoid allowing generic course creation without a capacity link.

Codex should not overbuild a complex external indicator engine in Phase 1. The first version can use a practical internal enum/list plus reference text fields.

when they complete the Analysis Gate,

when it is published,

then the course catalog and monitoring dashboard show MEAL as the capacity area.

when the achievement record is created,

then it links to the course, capacity area, indicator/standard, proof type, and verification date.
