# Course-Fit Decision Matrix

Use this matrix when planning, implementing, or reviewing Analysis Gate behavior.

| Evidence pattern | K/S/M/E classification | Course-fit decision | Design unlock? | Required note |
|---|---|---|---|---|
| Learners do not know a concept, standard, rule, process, or distinction | Knowledge | `course_addressable` | Yes, after handover lock | State what learners need to know and why it matters for CSO capacity. |
| Learners know the concept but cannot apply it in a practical task, tool, decision, or output | Skill | `course_addressable` | Yes, after handover lock | State the practical action learners must perform. |
| Staff know and can act but do not prioritize, value, trust, or feel ownership of the practice | Motivation | `not_course_addressable` | No | Route to complementary support; do not create Phase 1 course unless separable K/S exists. |
| Staff are blocked by missing tools, systems, policies, time, resources, approval, internet, civic-space conditions, or safety risks | Environment | `not_course_addressable` | No | Route to complementary support; do not create Phase 1 course unless separable K/S exists. |
| Evidence shows both K/S and M/E causes, and the K/S component is clear | Mixed | `partly_course_addressable` | Yes, only for K/S component after handover lock | Record K/S component separately from M/E components. |
| Evidence shows mixed causes but K/S component is unclear | Mixed / unclear | `needs_further_diagnosis` | No | Request further diagnosis before course design. |
| Evidence is weak, missing, contradictory, or not tied to a target learner group | Unknown | `needs_further_diagnosis` | No | Explain missing evidence and required clarification. |
| Advocacy, safeguarding, civic-space, legal/compliance, or sensitive data risk is present | Any | `specialist_review_required` | No or conditional | Require specialist review before unlock or before sensitive design work. |

## Required Analysis-to-Design Handover confirmation

Before Design unlocks, confirm:

- Capacity gap is specific and validated.
- Target learner group is defined.
- Capacity area and sub-capacity are recorded where available.
- Indicator or standard link is recorded where available.
- Current baseline/current practice is recorded.
- Desired practice is recorded.
- Root cause is recorded.
- K/S/M/E route is recorded.
- Course-fit decision is recorded.
- Separable K/S component is recorded for mixed gaps.
- M/E routing note is recorded where relevant.
- Safeguards/no-harm note is recorded.
- Evaluation anchor is recorded.
- Analysis Handover is approved or locked.

## Plain-language UI messages

Use messages like these where appropriate:

### Course-addressable

This gap can proceed to course design because the evidence shows a Knowledge or Skill gap that a course can realistically address.

### Partly course-addressable

This gap has several causes. The course can address the recorded Knowledge/Skill component, but the Motivation or Environment barriers should be handled through complementary support.

### Not course-addressable

This issue should not become a Phase 1 course because the main barrier is not Knowledge or Skill. Record it for complementary support instead.

### Needs further diagnosis

There is not enough evidence to decide whether a course is the right response. Clarify the root cause, target learner group, or separable Knowledge/Skill component before moving to Design.

### Specialist review required

This case may involve safeguarding, civic-space, legal, data protection, or other sensitive risks. A specialist review is needed before the course can move forward.
